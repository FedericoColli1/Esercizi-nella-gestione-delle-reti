Creazione di una rete formata da due bridge. Lo scopo principale dell'esercitazione è quello di creare un client, un server e impostare un composer che svolga la funzione di nodo di passaggio, per permettere ai due attori di comunicare. Questo è possibile abilitando su ns42 il passaggio di pacchetti e impostandolo come route default (o gateway) per client e server. In questo modo, ns42 svolgerà funzione di router tra le due reti create con l'utilizzo dei due bridge separati uno per il client e uno per il server. Ovviamente, ns42 è collegato a entrambe le reti.


- docker exec ns41 ip route
default via 10.10.10.1 dev eth0
10.10.10.0/24 dev eth0 proto kernel scope link src 10.10.10.41

default via 10.10.10.42 dev eth0
10.10.10.0/24 dev eth0 proto kernel scope link src 10.10.10.41


- docker exec ns42 ip route
default via 10.10.10.1 dev eth0
10.10.10.0/24 dev eth0 proto kernel scope link src 10.10.10.42
20.20.20.0/24 dev eth1 proto kernel scope link src 20.20.20.42

- docker exec ns43 ip route
default via 20.20.20.1 dev eth0
20.20.20.0/24 dev eth0 proto kernel scope link src 20.20.20.43

default via 20.20.20.42 dev eth0
